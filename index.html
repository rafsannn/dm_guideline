Question Paper Pattern Analysis

Phase 1 — Past Paper Insights

Repeated High-Weight Topics

◆Truth Table Construction — virtually every paper requires constructing tables for compound propositions or to prove logical equivalences.
◆English to Logic Translation — frequent questions translating complex English sentences into propositional or predicate logic.
◆Function Properties — defining and distinguishing injective, surjective, and bijective functions.
◆Sequences & Summations — finding formulas, AP/GP identification, and double summation calculations.
◆Set Operations & Cartesian Products — union, intersection, difference, and power sets.

Question Structure Types

▸Problem-solving exercises dominate — bit string counting, set operations, summation values.
▸Definitions & short explanations — "tautology," "cardinality," "sum rule" come up regularly for easy marks.
▸Proof-based questions — proofs done via truth tables, not formal induction.
▸C Program Implementation — a unique pattern: C programs using for loops to implement specific summations.
Previously Asked but Not Clearly Covered in Slides: Searching Algorithms (binary/linear search steps + pseudocode), Rules of Inference (Modus Ponens, Hypothetical Syllogism), Permutations & Combinations (proofs involving P(n,r) and C(n,r)).

05

Exam Strategy

Phase 5 — Win the Exam

01
⚡

Prioritize Problem Solving

The exam is heavily weighted toward calculating values — summations, set operations, bit strings — and constructing truth tables. Drill these types first.

02
🔺

Master the "Big Three" Proofs

Be prepared to prove: (1) Logical Equivalences via truth tables, (2) Set equalities, and (3) Function types (one-to-one / onto). These appear in every paper.

03
✍️

English Fluency in Logic

Practice translating "whenever," "is necessary for," and "unless" into conditional logic. The teacher uses these precise phrasings deliberately to test translation skill.

04
∑

Double Summation = Free Marks

These problems follow a very predictable pattern and carry significant marks. Also ensure you can describe the double-loop logic in a simple C for loop — easy additional points.

05
📖

Memorize Key Definitions

Definition questions like "Define Power Set" or "What is a Tautology?" are frequent and provide easy marks when memorized directly from the slides.

Quick Reminder: Topics previously asked but not clearly covered in slides — Binary/Linear Search, Modus Ponens / Rules of Inference, and P(n,r) / C(n,r) proofs — may appear again. If time permits, review these from a textbook for potential bonus marks.
